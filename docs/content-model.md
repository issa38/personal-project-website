# Content Model

This site is intentionally built so future proof points can be added without redesigning the portfolio.

## Case Study Template

- Title
- Status: draft, in progress, published, archived
- Business question
- Context and constraints
- Operating or financial value at stake
- Method used
- Technical approach
- Metrics or evidence
- Tradeoffs
- Outcome or expected outcome
- Next iteration

## Project Lab Template

- Title
- Status: concept, in development, shipped, archived
- Business thesis
- User or workflow served
- Tools and stack
- Strategic value
- Current maturity
- Demo, repository, screenshot, or note link
- What changed because of the project

## Notes Template

- Title
- Category: strategy, operations, finance, AI systems, builder log
- Core claim
- Practical example
- Business implication
- Follow-up project or case-study link

## Review Standard

A new content item should make at least one of these stronger:

- Hiring manager clarity
- Business acumen evidence
- Technical credibility
- Personal point of view
- Long-term portfolio depth
